## Context

当前 `general-server` 的 `siteMenu` 模块已经具备完整六层结构，并支持通过 `siteMenu.json` 初始化空表，也支持按节点进行 CRUD。但系统还没有“整包导入菜单文件”的能力，导致批量维护菜单时只能逐条新增或手工改库。  
现有菜单种子结构是树形 JSON 数组，节点主要字段为 `id`、`name`、`path`、`icon`、`children`。数据库层已经具备把树形结构拍平并重建菜单树的能力，因此文件导入最自然的方式是：接收上传 JSON -> 校验结构 -> 替换数据库菜单 -> 同步更新本地 `siteMenu.json`。

## Goals / Non-Goals

**Goals:**

- 为 `siteMenu` 增加一个语义化文件上传导入接口
- 只接受受控的菜单 JSON 文件，并校验根结构与节点字段
- 导入后同时更新 `sys_site_menu` 与 `siteMenu.json`
- 保持现有 `siteMenu` 模块的六层结构和中文返回契约
- 补齐上传导入相关的测试

**Non-Goals:**

- 本次不做前端上传页面
- 本次不支持 Excel、CSV 等非 JSON 菜单文件
- 本次不做部分合并导入，导入行为以“整体替换当前菜单”为准

## Decisions

### 决策 1：新增单一业务路由 `POST /api/site-menu/uploadMenuFile`

在现有 `siteMenuRouter` 中新增一个上传接口，命名为 `uploadMenuFile`，继续保持“一个业务域一个 router”的规则。  
选择这个动作式路径而不是通用 `/import` 或 `/upload` 的原因：

- 与当前仓库的动作式接口命名保持一致
- 能直接表达“这是菜单文件上传导入”而不是泛上传能力
- 不需要额外拆分子 router

### 决策 2：导入行为采用“整体替换”而不是增量合并

上传的菜单文件解析成功后，先清空现有菜单表，再按文件内容重建 `sys_site_menu`，并同步覆盖 `siteMenu.json`。  
这样做优于增量合并的原因：

- 当前种子结构天然就是整树表达，整体替换更符合文件语义
- 可以避免旧节点残留、排序冲突和父子关系不一致
- 集成测试更容易验证结果

备选方案是按节点差量合并，但需要定义冲突规则、删除策略和 id 对齐策略，复杂度显著更高，因此本次不采用。

### 决策 3：只支持 JSON 文件，并在 service 层完成解析校验与中文异常收口

controller 负责接收上传文件，service 负责：

- 校验文件是否存在
- 校验文件扩展名与 MIME 合法性
- 解析 JSON
- 校验根节点是否为数组以及节点结构是否合法
- 调用 repository 完成数据替换

这样做优于把解析逻辑放进 controller 的原因：

- 保持协议层与业务层职责分离
- 更容易补单元测试
- 能复用现有 `SiteMenuBusinessError` 返回中文受控错误

### 决策 4：引入 `multer` 处理 multipart 文件上传

当前后端没有 multipart 解析能力，因此新增 `multer` 作为上传中间件，采用内存缓冲读取文件内容。  
选择内存上传而不是先落磁盘的原因：

- 菜单 JSON 文件体量通常较小
- 不需要引入额外临时文件清理逻辑
- 更适合“解析后立即覆盖导入”的使用方式

## Risks / Trade-offs

- [风险] 上传文件格式不规范会导致导入失败
  → Mitigation：严格校验 JSON 根结构、字段类型与节点层级，并返回中文错误

- [风险] 导入是整体替换，误上传会覆盖当前菜单
  → Mitigation：仅在完整校验通过后执行替换，并保留 `siteMenu.json` 与数据库一致，便于后续回滚文件内容

- [风险] 使用内存上传对超大文件不友好
  → Mitigation：本次仅面向菜单 JSON 文件，后续可增加文件大小限制

## Migration Plan

1. 新增菜单文件上传接口与 DTO 契约
2. 引入上传中间件并在 controller/router 接入
3. 在 service/repository 中实现文件解析校验、整体替换和 `siteMenu.json` 同步
4. 补充单元测试与集成测试
5. 运行后端测试并确认上传后 `getMenu` 返回已更新

回滚策略：

- 若实现异常，可回滚本次接口与依赖改动
- 若数据导入错误，可用原始 `siteMenu.json` 重新执行一次导入恢复

## Open Questions

- 是否需要对上传文件大小设置显式上限；本次先按常规菜单 JSON 小文件假设实现
- 是否需要保留导入历史；本次不纳入范围
